### Insertion sort
> Un algoritmo semplice e intuitivo che costruisce la lista ordinata un elemento alla volta, inserendo ciascun nuovo elemento nella posizione corretta tra quelli gi√† ordinati. Funziona bene su array **piccoli** o **quasi ordinati**.  
> √à in-place e stabile, quindi adatto all‚Äôordinamento online e come caso base in algoritmi pi√π complessi come **Merge Sort** o **Quick Sort** in implementazioni ibride.

| $\textbf{Best}$ | $\textbf{Average}$ | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| --------------- | ------------------ | ---------------- | ------------------- | ----------------- |
| $\Omega (n)$    | $\Theta (n^2)$     | $O(n^2)$         | ‚úÖ                  | ‚úÖ                |

```C title:InsertionSort.c fold
void InsertionSort(int A[], int N) {
	const int l=0, r=N-1;
    int i, j, x;
    
    for (i = l+1; i <= r; i++) {
        x = A[i];
        j = i - 1;
        
        while (j >= l && x < A[j]) {
            A[j+1] = A[j];
            j--;
        }
        
        A[j+1] = x;
    }
}
```

### Exchange / Bubble sort
> Un algoritmo di base che passa ripetutamente attraverso la lista, **confronta gli elementi adiacenti** e **li scambia** se sono nell‚Äôordine sbagliato. Sebbene sia facile da implementare e comprendere, √® molto inefficiente su grandi dataset.  
> La variante ottimizzata introduce una flag per interrompere in anticipo se non vengono effettuati scambi in un passaggio. Tuttavia, rimane inadatto per compiti critici in termini di prestazioni.

| $\textbf{Best}$ | $\textbf{Average}$ | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| --------------- | ------------------ | ---------------- | ------------------- | ----------------- |
| $\Omega (n)$    | $\Theta (n^2)$     | $O(n^2)$         | ‚úÖ                  | ‚úÖ                |

```C title:BubbleSort.c fold
void BubbleSort(int A[], int N) {
	const int l=0, r=N-1;
    int i, j;
    int temp;
    
    for (i = l; i < r; i++) {
        for (j = l; j < r - i + l; j++) {
            if (A[j] > A[j+1]) {
                temp = A[j];
                A[j] = A[j+1];
                A[j+1] = temp;
            }
        }
    }
    return;
}
```

### Selection sort
> Un algoritmo semplice basato su confronti che seleziona ripetutamente **l‚Äôelemento minimo** dalla parte non ordinata e lo **porta all‚Äôinizio**. Esegue il numero minimo di scambi $O(n)$, ma il numero quadratico di confronti lo rende inefficiente su array grandi.  
> √à in-place ma non stabile, il che ne limita l‚Äôuso pratico.

| $\textbf{Best}$ | $\textbf{Average}$ | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| --------------- | ------------------ | ---------------- | ------------------- | ----------------- |
| $\Omega (n^2)$  | $\Theta (n^2)$     | $O(n^2)$         | ‚úÖ                  | ‚ùå                |

```C title:SelectionSort.c fold
void SelectionSort(int A[], int N) {  
    const int l=0, r=N-1;
    int i, j, min;
    int temp;
    
    for (i = l; i < r; i++) {
        min = i;
        
        for (j = i+1; j <= r; j++) {
            if (A[j] < A[min])
                min = j;
        }
        
        if (min != i) {
            temp = A[i];
            A[i] = A[min];
            A[min] = temp;
        }
    }
    return;
}
```

### Shell sort
> Una generalizzazione efficiente dell‚Äô**Insertion Sort** che permette di **confrontare e scambiare elementi lontani tra loro**. Anche se non √® stabile, √® in-place e **funziona bene su dataset di medie dimensioni**.

| $\textbf{Best}$      | $\textbf{Average}$     | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| -------------------- | ---------------------- | ---------------- | ------------------- | ----------------- |
| $\Omega (n \log n)$* | $\Theta (n \log^2 n)$* | $O(n^2)$*        | ‚úÖ                  | ‚ùå                |

> [!NOTE]
> La sua complessit√† dipende dalla sequenza di intervalli (gap) usata.  
> La sequenza proposta da Knuth √®:  
> $$h_k = \frac{3^k - 1}{2} \quad \quad (1,\ 4,\ 13,\ 40,\ 121,\ 364,\ \ldots)$$  
> Nel caso peggiore, pu√≤ arrivare a $O(n^{3/2})$.

> [!NOTE]
> $$
\begin{array}{111}
\textbf{Shell} && 1, \ 2, \ 4, \ 8, \ 16 \dots && O(n^2) \\
\textbf{Hibbard} && 1, \ 3, \ 7, \ 15, \ 31 \dots && O(n^{3/2}) \\
\textbf{Pratt} && 1, \ 2, \ 3, \ 4, \ 6, \ 8, \ 9, \ 12 \dots && O(n \log^2{n}) \\
\textbf{Knuth} && 1, \ 4, \ 13, \ 40, \ 121 \dots && O(n^{3/2}) \\
\textbf{Sedgewick} && 1, \ 5, \ 19, \ 41, \ 109, \ 209 \dots && O(n^{4/3})
\end{array}$$

```C title:ShellSort.c fold
void ShellSort(int A[], int N) {
	const int l=0, r=N-1;
    int i, j, x, h=1;
    
    while (h <= N/3)
        h = 3*h+1;  // sequenza di Knuth
    
    while (h >= 1) {
        for (i = l + h; i <= r; i++) {
            j = i;
            x = A[i];
            
            while (j >= l + h  && x < A[j-h]) {
                A[j] = A[j-h];
                j -=h;
            }
            
            A[j] = x;
        }
        h = h/3;
    }
}
```

### Counting sort
> Un algoritmo di ordinamento che **non si basa su confronti**, ma conta il numero di occorrenze di ogni valore unico. √à ideale per insiemi di interi **con intervalli piccoli**, dove il valore massimo $k$ non √® molto pi√π grande del numero di elementi $n$.  
> Ha complessit√† lineare, ma **non √® in-place** e usa spazio extra $O(k)$. √à stabile e spesso usato nel [[#Radix sort]].

| $\textbf{Best}$  | $\textbf{Average}$ | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| ---------------- | ------------------ | ---------------- | ------------------- | ----------------- |
| $\Omega (n + k)$ | $\Theta (n + k)$   | $O(n + k)$       | ‚ùå                  | ‚úÖ                |

```C title:CountingSort.c fold
void CountingSort(int A[], int B[], int C[], int N, int k) {  
    const int l=0, r=N-1;
    int i;
    
    for (i = 0; i < k; i++)
        C[i] = 0;  // inizializzazione di C
        
    for (i = l; i <= r; i++)
        C[A[i]]++;  // occorrenze semplici
        
    for (i = 1; i < k; i++)
        C[i] += C[i-1];  // occorrenze multiple
        
    for (i = r; i >= l; i--) {  // ordinamento
        B[C[A[i]] - 1] = A[i];
        C[A[i]]--;
    }
    
    for (i = l; i <= r; i++)
        A[i] = B[i];  // ricopiatura risultato
}
```

### Radix sort
> Un algoritmo di ordinamento non basato su confronti che lavora **cifra per cifra**, spesso su interi o stringhe di lunghezza fissa. **Processa le cifre dalla meno significativa alla pi√π significativa**, spesso usando il [[#Counting Sort]] come passo intermedio stabile.  
> Ha complessit√† lineare quando il numero di cifre $k$ √® costante o piccolo rispetto a $n$, ma richiede spazio extra e **non √® in-place**. Funziona **molto bene su insiemi grandi** con chiavi limitate.

| $\textbf{Best}$ | $\textbf{Average}$ | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| --------------- | ------------------ | ---------------- | ------------------- | ----------------- |
| $\Omega (nk)$   | $\Theta (nk)$      | $O(nk)$          | ‚ùå                  | ‚úÖ                |


```C title:CountingSortForRadix.c fold
    const int l=0, r=N-1;
    int i, weight=1;
	
    for (i = 0; i < step; i++)
	    weight *= 10;  // calcolo di 10^step
		
    for (i = 0; i < 10; i++)
	    C[i] = 0;
		
    for (i = l; i <= r; i++)
	    D[i] = (A[i] / weight) % 10;  // identificazione colonna
		
    for (i = l; i <= r; i++)
	    C[D[i]]++;  // occorrenze semplici
		
    for (i = 1; i < 10; i++)
	    C[i] += C[i - 1];  // occorrenze multiple
		
    for (i = r; i >= l; i--) {  // ordinamento
        B[C[D[i]] - 1] = A[i];
        C[D[i]]--;
    }
	
    for (i = l; i <= r; i++)
	    A[i] = B[i];  // ricopiatura
}
```

### Bottom-up Merge sort
> Una versione **iterativa** del Merge Sort che elimina la ricorsione unendo progressivamente **sotto-array adiacenti** di dimensione crescente (1, 2, 4, 8, ...).  
> Come la sua controparte ricorsiva, garantisce un tempo di esecuzione di $O(n \log n)$ in tutti i casi ed √® **stabile**. Tuttavia, richiede **$O(n)$ di spazio extra** e non √® in-place. √à particolarmente adatto per **linked lists** e **external sorting**.

| $\textbf{Best}$     | $\textbf{Average}$  | $\textbf{Worst}$ | $\textbf{In-place}$ | $\textbf{Stable}$ |
| ------------------- | ------------------- | ---------------- | ------------------- | ----------------- |
| $\Omega (n \log n)$ | $\Theta (n \log n)$ | $O(n \log n)$    | ‚ùå                  | ‚úÖ                |

```C title:BottomUpMergeSort.c fold
void BottomUpMergeSort(int A[], int B[], int N) {  // B vettore ausiliario  
    const int l=0, r=N-1;
    int i, q, m;
    
    for (m = 1; m <= r - l; m = m + m) {
        for (i = l; i <= r - m; i += m + m) {
	        // identificazione dell‚Äôindice iniziale della prossima coppia
	        // di sottovettori contigui e ordinati di dimensione m
            q = i+m-1;
            
            // fusione di coppie di sottovettori ordinati
            Merge(A, B, i, q, r, m);
        }
    }
}
```

```C title:Merge.c fold
void Merge(int A[], int B[], int l, int q, int r, int m) {  
    int i, j, k;
    i = l;
    j = q+1;
    
    for (k = l; k <= r; k++) {
        if (i > q)  // esaurito sottovettore sinistro  
            B[k] = A[j++];
        else if (j > q+m)  // esaurito sottovettore destro  
            B[k] = A[i++];
        else if ((A[i] < A[j]) || (A[i] == A[j]))
            B[k] = A[i++];
        else
	    B[k] = A[j++];
    }
	        
    for ( k = l; k <= r; k++ )
        A[k] = B[k];
        
    return;  
}
```
